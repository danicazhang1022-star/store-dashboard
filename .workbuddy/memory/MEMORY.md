## 每日销售表录入方式
- **每件衣服一行**（不使用多选关联），查找引用字段正常工作
- **订单号字段**：同一单多件衣服填同一个订单号，用于大单奖励汇总计算
- 大单奖励逻辑：按订单号汇总金额 ≥1000→¥10, ≥2000→¥20, 每人每天最多计2笔
- 日期字段设默认值"今天"，新增记录自动填当天日期
- 用户每天操作：在腾讯文档智能表格录入 → 销售员 → 订单号 → 实收金额

## 重要操作规则
- **不得修改已有数据**：限流恢复后读取每日销售表，已有数据只读不改
- **不得修改表格行列顺序**：不调整字段顺序、不增删行

## 腾讯文档 MCP 关键技术点
- **图片字段(field_type=5) 写入格式**：`[{"image_id": "xxx"}]`（JSON数组），不是纯字符串
- **upload_image 返回的 image_id 有效期仅1天**，上传和写入必须在同一会话连续完成
- API限流错误码 400007，频繁调用会触发
- 工作表名称通过API创建可能出现中文乱码，建议手动在界面重命名
- 关联字段需手动在界面设置（删除文本列→新建关联列→添加查找引用列）
